What This Tool Does
This tool automates the generation of individual bank upload files for every employee in your monthly payroll cycle. You upload a single Employee Salary Excel (containing Emp ID, Name, and Net Amount). The tool then looks up each employee in the embedded Master Saral Bank database (1,470 records) to fetch their Account Number, Bank Name, and IFSC Code. Based on the bank, it routes each employee to either a 275TSAL file (HDFC internal transfer) or a 275TNES file (NEFT to other banks). One file is created per valid employee, each with a unique serial number, and all files are packaged into a single ZIP for bank submission.
Standard Operating Procedure (SOP)
1
Prepare the Salary File — Export from Saral Payroll with columns: SNo Emp ID Emp Name Amount. Ensure Amount is the final Net Pay. Remove rows with zero or blank amounts before upload.
2
Upload the Salary File — Click Browse File or drag-and-drop the .xlsx file. The tool auto-detects column headers. Verify the column mapping shown under Step 2.
3
Set Parameters — Enter the Payment Date (used in filename as DDMM), the Last Serial No. used in the previous run (starts at 0 for first run of the month), and the Value Date in YYYYMMDD format for NEFT files.
4
Generate Files — Click Generate Employee Files. Each valid employee gets one file: 275TSAL for HDFC employees (internal credit) and 275TNES for NEFT employees (other banks). Serial numbers increment by 1 per file across both types.
5
Review Discrepancy Report — Check for warnings (zero pay, blank rows) and errors (employees not found in Master DB, missing IFSC or Account No). Fix the Master Saral database for flagged employees and re-run if needed.
6
Download & Submit — Click Download All as ZIP to get all individual files in one package. Submit 275TSAL* files to HDFC NetBanking bulk upload portal and 275TNES* files to the NEFT bulk payment portal. Note the next serial number for the following run.
